RF Based Communication System in Disaster Zones

Major Project Phase-1 — Ballari Institute of Technology & Management

About Team Objectives Design Hardware References

Abstract & Purpose

This project focuses on development and application of RF-based communication systems for disaster zones, where traditional infrastructure may fail. Systems such as LoRa, MANETs and Delay Tolerant Networks are evaluated for suitability. The design emphasises portable, energy-efficient hardware, adaptive routing protocols and hybrid relays (e.g., UAVs) for resilient communication. (Source: project report). 1

Purpose

  • Explore RF wireless communication principles and build an infrastructure-independent communication system.
  • Provide a comprehensive record of objectives, methods, challenges and conclusions.

Objectives

  • Design an infrastructure-independent communication system for disaster-affected regions.
  • Implement an RF-based wireless framework for real-time emergency data transmission.
  • Develop a portable, cost-effective, energy-efficient communication device.
  • Create a scalable and modular network solution for flexible deployment.
  • Ensure secure and reliable communication with error-checking and validation.
  • Provide a simple interface suitable for non-technical users.
  • Test and validate under simulated disaster scenarios.

Design & Methodology

The methodology follows requirement analysis → architecture & component selection → prototype development → integration → testing under simulated conditions → optimization. The design principles include modular architecture, mesh networking topology, energy-efficient operation, rugged hardware, integrated GPS & alert systems, flexible power, simple user interface and reliable communication protocols. 2

System Architecture (summary)

  • Modular separation: communication, power, control
  • Mesh network for multi-hop redundancy
  • Low-power modules (LoRa, NRF24L01) and microcontrollers (ESP32/Arduino)
  • GPS integration, OLED displays, buttons for local interaction
  • Support for battery + solar charging

Hardware Requirements

  • Microcontroller — Arduino Nano/Uno or ESP32
  • RF Modules — NRF24L01+ or LoRa SX1278
  • OLED/LED Display to show messages
  • Keypad / Push Buttons for input
  • Battery & Power Module (and optional solar)
  • Rugged enclosure for field deployment

Functional / Non-functional Highlights

Real-time short/long-range messaging, portable low-power nodes, mesh or star networking, simple UI, cost-effective prototype.
